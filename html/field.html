Fields¶

NField¶

class treecorr.NField(cat, min_sep, max_sep, b, split_method='mean', metric='Euclidean', max_top=10, logger=None)¶

This class stores the positions in a tree structure from which it is efficient to compute the two-point correlation functions.

An NField is typically created from a Catalog object using

>>> nfield = cat.getNField(min_sep, max_sep, b)
Parameters:
  • cat – The catalog from which to make the field.
  • min_sep – The minimum separation between points that will be needed.
  • max_sep – The maximum separation between points that will be needed.
  • b – The b parameter that will be used for the correlation function. This should be bin_size * bin_slop.
  • split_method – Which split method to use (‘mean’, ‘median’, or ‘middle’) (default: ‘mean’)
  • metric –

    Which metric to use for distance measurements. Options are:

    • ‘Euclidean’ = straight line Euclidean distance between two points. For spherical coordinates (ra,dec without r), this is the chord distance between points on the unit sphere.
    • ‘Rperp’ = the perpendicular component of the distance. For two points with distance from Earth r1, r2, if d is the normal Euclidean distance and \(Rparallel = |r1-r2|\), then we define \(Rperp^2 = d^2 - Rparallel^2\).

    (default: ‘Euclidean’)

  • max_top – The maximum number of top layers to use when setting up the field. (default: 10)
  • logger – A logger file if desired (default: None)
nTopLevelNodes¶

The number of top-level nodes.
